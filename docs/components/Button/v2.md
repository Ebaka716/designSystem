# Button V2: Consolidated Standard & Conversational Button\n\nThis is the primary button component, consolidating features from previous versions (V1-V4). It supports standard variants (`default`, `outline`, `ghost`), a distinct `conversational` variant, sizing, icons, loading states, and fill styles for the conversational variant.\n\n## Description\n\nA versatile button component suitable for various actions, from primary calls-to-action to secondary/tertiary options and conversational flows.\n\n## Props\n\nInherits standard HTML `<button>` attributes.\n\n| Prop           | Type                                                    | Default     | Description                                                                                                    |\n| :------------- | :------------------------------------------------------ | :---------- | :------------------------------------------------------------------------------------------------------------- |\n| `variant`      | `'default'` \| `'outline'` \| `'ghost'` \| `'conversational'` | `'default'` | The visual style of the button. `conversational` uses a distinct theme and unique bottom-left rounding.        |\n| `size`         | `'sm'` \| `'default'` \| `'lg'` \| `'icon'`              | `'default'` | Button size. `icon` size is specifically for buttons containing only an icon (fixed width/height, no padding). |\n| `fill`         | `'solid'` \| `'outline'` \| `'ghost'`               | `'solid'`   | Fill style, primarily for the `conversational` variant. Controls background/border. Ignored by other variants. |\n| `icon`         | `React.ReactNode`                                       | `undefined` | An optional icon element to display within the button.                                                         |\n| `iconPosition` | `'leading'` \| `'trailing'`                        | `'leading'` | Position of the icon (or loader) relative to the button text (`children`).                                     |\n| `loading`      | `boolean`                                               | `false`     | If true, displays a loading spinner instead of an icon/children and disables the button.                     |\n| `asChild`      | `boolean`                                               | `false`     | Render as a child element (e.g., `<Slot>`), inheriting props. Notes\n- Uses `cva` (class-variance-authority) for managing variants (`variant`, `size`, `fill`, `iconPosition`, internal `hasIcon`).\n- Supports `default`, `outline`, `ghost`, and `conversational` variants.\n- The `conversational` variant uses a distinct teal color scheme and specific rounding (`rounded-md rounded-bl-none`), controlled primarily by the `fill` prop (`solid`, `outline`, `ghost`).\n- Handles icon rendering (`icon` prop) and placement (`iconPosition` prop: `leading` (default) or `trailing`).\n- Handles loading state (`loading` prop), displaying a spinner and disabling the button. The spinner replaces the icon if both are potentially present.\n- Children are hidden when `loading` is true.\n- Uses `compoundVariants` for adjusting icon padding based on size/position and for applying conversational variant styles (rounding, colors based on fill).\n- Includes an `icon` size variant for icon-only buttons, which removes padding and sets a fixed size (`h-9 w-9`).\n
